Linux 安全合规审计脚本

Python3 通用Linux

全面扫描Linux系统安全配置状态，生成HTML格式合规报告，涵盖文件权限、网络配置、用户账户、SSH配置、PAM策略等120+检查项，支持CIS等保2.0双标准映射。

Python 
#!/usr/bin/env python3
# IHCCC Linux 安全审计脚本
# 生成CIS/等保2.0合规报告

import subprocess, os, json
from datetime import datetime

CHECKS = {
    "SSH_ROOT_LOGIN": {
        "cmd": "grep '^PermitRootLogin' /etc/ssh/sshd_config",
        "expected": "PermitRootLogin no",
        "cis_id": "5.2.10",
        "level": 1
    },
    "PASSWD_MAX_DAYS": {
        "cmd": "grep '^PASS_MAX_DAYS' /etc/login.defs",
        "expected": "PASS_MAX_DAYS 90",
        "cis_id": "5.4.1.1",
        "level": 1
    }
}

def run_check(name, check):
    result = subprocess.run(check["cmd"], shell=True,
                            capture_output=True, text=True)
    passed = check["expected"] in result.stdout
    return {"name": name, "passed": passed,
            "actual": result.stdout.strip()}

# 运行所有检查并输出报告
results = [run_check(n, c) for n, c in CHECKS.items()]
passed = sum(1 for r in results if r["passed"])
print(f"合规检查完成: {passed}/{len(results)} 项通过")

Bash 
#!/bin/bash
# IHCCC 等保2.0三级 Linux自检脚本
# 覆盖GB/T 22239-2019技术要求

PASS=0; FAIL=0; WARN=0

check() {
  local desc=$1 cmd=$2
  if eval "$cmd" >/dev/null; then
    echo "[✓] $desc"; ((PASS++))
  else
    echo "[✗] $desc"; ((FAIL++))
  fi
}

echo "=== 身份鉴别检查 ==="
check "禁止空密码登录" \
  "grep -q 'PermitEmptyPasswords no' /etc/ssh/sshd_config"

  "grep -q 'ClientAliveInterval' /etc/ssh/sshd_config"

  "awk '/^PASS_MAX_DAYS/{print \$2}' /etc/login.defs | \
   awk '\$1<=90{exit 0}{exit 1}'"

check "auditd服务运行中" \
  "systemctl is-active --quiet auditd"
check "记录sudo操作日志" \
  "grep -q 'sudo' /etc/audit/rules.d/*.rules 2>/dev/null"

echo "汇总：通过 $PASS | 失败 $FAIL | 警告 $WARN"

Windows 安全基线检测报告

PowerShell 合规报告

自动检测Windows系统安全配置并生成HTML格式合规报告，涵盖系统配置、账户安全、网络安全、注册表安全设置等80+检查项，支持与CIS基准对照输出通过率。

PowerShell 
# IHCCC Windows 安全检测脚本
# 输出合规检查报告

function Check-SecurityItem {
    param($Name, $ScriptBlock, $Expected, $CisID)
    $actual = & $ScriptBlock
    $pass = $actual -eq $Expected
    [PSCustomObject]@{
        CisID   = $CisID
        Name    = $Name
        Status  = if ($pass) { "通过" } else { "失败" }
        Actual  = $actual
        Expected = $Expected
    }
}

$report = @(
    Check-SecurityItem "最短密码长度" {
        (net accounts | Select-String "Minimum password length") -replace "\D", ""
    } "14" "1.1.4",

    Check-SecurityItem "防火墙状态(域)" {
        (Get-NetFirewallProfile -Name Domain).Enabled
    } "True" "9.1.1"
)

$pass = ($report | Where-Object Status -eq "通过").Count
Write-Host "合规率: $pass/$($report.Count)"
$report | Export-Csv "cis-report-$(Get-Date -f yyyyMMdd).csv" -Encoding UTF8

Docker 安全审计脚本

Bash Docker

基于CIS Docker Benchmark的安全审计脚本，检测Docker主机配置、Docker守护进程安全设置、容器运行时配置、镜像安全、网络配置等方面的合规状态，生成详细审计报告。

Bash 
#!/bin/bash
# IHCCC Docker 安全审计脚本
# 基于 CIS Docker Benchmark

PASS=0; FAIL=0

cis_check() {
  local id=$1 desc=$2 cmd=$3
  if eval "$cmd" >/dev/null 2>&1; then
    echo "[PASS] $id: $desc"; ((PASS++))
  else
    echo "[FAIL] $id: $desc"; ((FAIL++))
  fi
}

echo "=== Docker守护进程配置检查 ==="

# 2.1 限制容器与容器间通信
cis_check "2.1" "禁用容器间通信" \
  "docker info --format '{{.DriverStatus}}' | grep -q icc=false"

# 2.2 设置日志级别
cis_check "2.2" "日志级别为info" \
  "docker info --format '{{.LoggingDriver}}' | grep -qv none"

# 2.5 启用内容信任
cis_check "2.5" "Docker内容信任已启用" \
  "[[ \$DOCKER_CONTENT_TRUST -eq 1 ]]"

# 检查所有运行中容器的配置
for cid in $(docker ps -q); do
  cis_check "5.1" "容器[$cid]非特权运行" \
    "docker inspect $cid --format '{{.HostConfig.Privileged}}' | grep -q false"
done

echo "汇总: 通过 $PASS | 失败 $FAIL"

Bash 
#!/bin/bash
# IHCCC Kubernetes CIS 合规检测脚本
# 使用 kube-bench 工具

KUBEBENCH_VERSION="v0.7.3"
REPORT_DIR="/tmp/k8s-cis-report-$(date +%Y%m%d)"

mkdir -p "$REPORT_DIR"

# 安装 kube-bench（如未安装）
if ! command -v kube-bench >/dev/null; then
  ARCH=$(uname -m | sed s/x86_64/amd64/ | sed s/aarch64/arm64/)
  URL="https://github.com/aquasecurity/kube-bench/releases/download"
  curl -L "$URL/$KUBEBENCH_VERSION/kube-bench_${KUBEBENCH_VERSION#v}_linux_$ARCH.tar.gz" \

fi

echo "[*] 正在检测 Kubernetes 控制平面..."
kube-bench run --targets master \
  --json > "$REPORT_DIR/master-report.json"

echo "[*] 正在检测工作节点..."
kube-bench run --targets node \
  --json > "$REPORT_DIR/node-report.json"

# 提取汇总信息
jq '[.Controls[].tests[].results[] |
  select(.status == "FAIL")] | length' \
  "$REPORT_DIR/master-report.json"
echo "[✓] 报告已保存至: $REPORT_DIR"

Redis服务安全加固脚本，检测并修复常见Redis安全问题：未授权访问、监听公网IP、危险命令（FLUSHALL/CONFIG/DEBUG）未禁用、持久化文件权限、慢查询日志未开启等，降低Redis被攻击风险。

Bash · Redis 
#!/bin/bash
# IHCCC Redis 安全加固脚本

REDIS_CONF="/etc/redis/redis.conf"

# 1. 绑定本地地址（防止公网暴露）
sed -i 's/^bind .*/bind 127.0.0.1/' $REDIS_CONF
echo "[✓] Redis绑定127.0.0.1"

# 2. 禁用危险命令（重命名为空字符串）
for cmd in FLUSHALL FLUSHDB CONFIG DEBUG SLAVEOF; do
  grep -q "rename-command $cmd" $REDIS_CONF ||     echo "rename-command $cmd """ >> $REDIS_CONF
done
echo "[✓] 危险命令已禁用"

# 3. 设置强密码（requirepass）
PASS=$(openssl rand -base64 32)
sed -i "s/^# requirepass.*/requirepass $PASS/" $REDIS_CONF
echo "[✓] 已设置随机密码，请保存: $PASS"

# 4. 检查是否存在未授权访问
if redis-cli ping 2>/dev/null | grep -q PONG; then
  echo "⚠️  Redis当前可无密码访问，请立即修复！"
fi

#!/bin/bash

ISSUES=0

check_fw() {

  if eval "$cmd"; then
    echo "[✓] $desc"

    echo "[✗] $desc"; ((ISSUES++))

}

echo "=== INPUT链默认策略 ==="

    echo "⚠️  端口 $port 对全网开放，建议限制来源IP"
    ((ISSUES++))
  fi
done

iptables -L -n -v --line-numbers

系统弱密码检测脚本

Python3 弱密码

检测系统用户密码强度，基于常见弱密码字典和密码复杂度规则，识别使用简单密码、空密码或默认密码的账户，同时检测SSH公钥认证配置，输出安全加固建议。

Python3 · 弱密码检测 
#!/usr/bin/env python3
# IHCCC 弱密码检测脚本

import subprocess, re

WEAK_PASSWORDS = [
    'password', '123456', 'admin', 'root',
    'qwerty', 'abc123', '111111', 'admin123'
]

def get_system_users():
    """获取有效的系统用户"""
    users = []
    with open('/etc/passwd') as f:
        for line in f:
            parts = line.strip().split(':')
            if len(parts) >= 7 and parts[6] not in ['/sbin/nologin', '/bin/false']:
                uid = int(parts[2])
                if uid == 0 or uid >= 1000:
                    users.append(parts[0])
    return users

def check_empty_password(user):
    """检查空密码"""
    result = subprocess.run(['passwd', '-S', user],
                            capture_output=True, text=True)
    return 'NP' in result.stdout  # NP = No Password

for user in get_system_users():
    if check_empty_password(user):
        print(f"⚠️  用户 [{user}] 存在空密码！")
    else:
        print(f"[✓] 用户 [{user}] 已设置密码")

Web 访问日志安全分析脚本

Python3 Nginx/Apache

分析Nginx/Apache访问日志，自动识别SQL注入、XSS、目录遍历、扫描器等Web攻击特征，统计高频请求IP、异常状态码分布，生成HTML格式安全分析报告。

Python3 · Web日志分析 
#!/usr/bin/env python3
# IHCCC Web 访问日志安全分析脚本

import re, sys
from collections import Counter

# 攻击特征规则
ATTACK_PATTERNS = {
    'SQL注入':     re.compile(r"union.+select|or\s+1=1|'--", re.I),
    'XSS攻击':     re.compile(r"<script|javascript:|onerror=", re.I),
    '目录遍历':    re.compile(r"\.\./|etc/passwd|/proc/self"),
    '扫描器探测':  re.compile(r"(sqlmap|nikto|nmap|masscan)", re.I),
    '命令注入':    re.compile(r";(cat|ls|id|whoami|wget)", re.I),
}

LOG_PATTERN = re.compile(
    r'(\S+) .+ \[.+\] "(\S+) (\S+) \S+" (\d+)'
)

results = Counter()
log_file = sys.argv[1] if len(sys.argv) > 1 else "/var/log/nginx/access.log"

with open(log_file) as f:
    for line in f:
        m = LOG_PATTERN.match(line)
        if not m: continue
        ip, method, uri, status = m.groups()
        for attack_type, pattern in ATTACK_PATTERNS.items():
            if pattern.search(uri):
                results[f"{ip} | {attack_type}"] += 1

for key, count in results.most_common(20):
    print(f"⚠️  [{count:4d}次] {key}")

$results = @()

function Check-Item($domain, $desc, $block) {
    $pass = try { & $block; $true } catch { $false }
    $results += [PSCustomObject]@{
        Domain = $domain; Desc = $desc
        Status = if ($pass) { "✓ 通过" } else { "✗ 不符" }
    }
}

# 身份鉴别
Check-Item "身份鉴别" "最短密码长度≥8位" {
    $len = (net accounts | Select-String "Minimum password length") -replace "\D",""
    if ([int]$len -lt 8) { throw }
}

# 安全审计
Check-Item "安全审计" "已启用登录成功审计" {
    $r = auditpol /get /category:*
    if ($r -notmatch "Logon.*Success") { throw }
}

# 入侵防范
Check-Item "入侵防范" "Windows防火墙已启用" {
    $fw = Get-NetFirewallProfile | Where-Object Enabled -eq $false
    if ($fw) { throw }
}

$results | Format-Table -AutoSize
$pass = ($results | Where-Object {$_.Status -like "*通过*"}).Count
Write-Host "合规率: $pass/$($results.Count)"
